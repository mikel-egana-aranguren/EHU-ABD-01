OWL: Web Ontology Language

Estándar oficial del W3C para crear ontologías en la web con un semántica precisa y formal

OWL

Se basa en Lógica Descriptiva (DL) para representar computacionalmente un dominio de conocimiento:

  • Razonamiento automático: inferir conocimiento "nuevo" (*), consultas, consistencia, clasificar entidades contra la ontología, ...
  • Integrar información dispersa mediante un vocabulario común

OWL

No es un lenguaje esquema con restricciones, se basa en inferencia (Para eso esta SHACL)

El mismo lenguaje RDF para definir datos y su vocabulario* (NoSQL!RDF!)

Sintaxis RDF/XML

Manchester OWL Syntax

Semántica OWL

Entidades: las entidades del dominio de conocimiento, identificadas con URIs, introducidas por el desarrollador ("Mikel", "participa_en", ...)

Axiomas: relacionan las entidades mediante el vocabulario lógico que ofrece OWL (namespace OWL)

Una ontología puede importar otra (owl:import) y hacer referencia a sus entidades mediante axiomas

OWL 2: Perfiles

OWL 2 define sublenguajes con diferentes trade-offs expresividad/complejidad:

OWL 2 EL: para ontologías grandes (millones de clases). Razonamiento en tiempo polinomial

OWL 2 QL: optimizado para consultas sobre datos. Reescritura de queries

OWL 2 RL: implementable con motores de reglas. Razonamiento eficiente

OWL 2 DL: expresividad completa, razonamiento decidible

OWL 2 EL

Usado en ontologías biomédicas: SNOMED CT (>300,000 clases)

Soporta: SubClassOf, equivalentClass, existencial (some)

No soporta: universal (only), negación, disyunción

Complejidad: PTime para razonamiento de subsunción

Razonador: ELK

OWL 2 QL

Basado en DL-Lite: query rewriting sobre bases de datos relacionales

Permite consultar BBDD relacionales virtualizadas como RDF

No soporta: existencial en lado derecho, restricciones de cardinalidad

Complejidad: LogSpace para razonamiento

Herramienta: Ontop (OBDA)

OWL 2 RL

Subset implementable con reglas (forward chaining)

Compatible con sistemas de producción basados en reglas

Soporta la mayoría de constructores OWL pero con restricciones

Complejidad: PTime

GraphDB usa reglas para implementar razonamiento OWL-Horst (similar a RL)

Lógica Descriptiva: DL

OWL se basa en familias de lógicas descriptivas

DL: fragmentos decidibles de lógica de primer orden

Nomenclatura: SROIQ(D) - OWL 2 DL

  • S: extensión de ALC con transitividad
  • R: propiedades complejas, reflexividad
  • O: nominals (individuos en expresiones de clase)
  • I: propiedades inversas
  • Q: restricciones de cardinalidad cualificadas
  • (D): datatypes

Ontology Design Patterns

Value Partition - Ejemplo

Class: Size
    SubClassOf: owl:Thing
    DisjointUnionOf: Small, Medium, Large

Class: TShirt
    SubClassOf: hasSize some Size

Class: LargeTShirt
    EquivalentTo: TShirt and (hasSize value Large)

Open World Assumption (OWA)

La ausencia de información no implica falsedad

Si no sabemos que Mikel vive en Bilbao, no asumimos que vive en otro sitio

Diferente de bases de datos relacionales (Closed World)

Consecuencia: las restricciones universales no son restrictivas por defecto

Para "cerrar" el mundo: usar propiedades funcionales, closure axioms, SHACL

No Unique Name Assumption (NUNA)

Dos URIs diferentes pueden referirse a la misma entidad

ex:Bilbao y ex:Bilbo pueden ser owl:sameAs

Útil para integración de datos de diferentes fuentes

Permite reconciliación automática mediante razonamiento

owl:differentFrom para forzar distinción

Anotaciones OWL

Metadatos sobre la ontología y sus entidades:

rdfs:label, rdfs:comment: etiquetas y descripciones

dc:creator, dc:date: Dublin Core para autoría

owl:versionInfo, owl:priorVersion: versionado

rdfs:seeAlso, rdfs:isDefinedBy: referencias

Pueden ser multilingües con language tags

Inconsistencias en OWL

Ontología inconsistente: contiene contradicciones lógicas

Clase insatisfacible: no puede tener instancias (equivalente a owl:Nothing)

Causas comunes:

  • Clases disjuntas con instancias compartidas
  • Restricciones de cardinalidad contradictorias
  • Propiedades funcionales con múltiples valores

Usar razonador para detectar y explicar inconsistencias

Explicaciones de inferencias

Justification: conjunto mínimo de axiomas que causa una inferencia

Herramientas: Protégé plugin de explicación

Debugging: identificar axiomas problemáticos

Puede haber múltiples justificaciones para una inferencia

Punning en OWL 2

Permitir que la misma URI sea clase, propiedad e individuo simultáneamente

Útil para metamodelado: clases que son instancias de otras clases

Ejemplo: ex:Mammal rdf:type owl:Class y ex:Mammal rdf:type ex:BiologicalCategory

Semántica: tratadas como entidades separadas según contexto

Property Chains

Definir propiedades complejas mediante composición

ObjectPropertyChain: 
    hasPadre o hasPadre → hasAbuelo

ObjectPropertyChain:
    localizadoEn o localizadoEn → localizadoEn
    (transitividad indirecta)

Inferir relaciones derivadas automáticamente

Keys en OWL 2

Identificar individuos únicamente mediante combinación de propiedades

Class: Person
    HasKey: hasSSN
    
Class: Publication
    HasKey: hasISBN, hasEdition

Similar a claves primarias en bases de datos relacionales

Permite inferir owl:sameAs automáticamente

Anotaciones: documentación adecuada (labels, comments)
