MongoDB

Base de datos NoSQL orientada a documentos

Los datos se almacenan como documentos JSON

No requiere un esquema fijo (schemaless)

MongoDB

Permite modelar la información alrededor de entidades reales en documentos JSON/BSON

Permite estructuras anidadas y arrays dentro del propio documento

Es muy útil cuando el esquema evoluciona con rapidez

Ideal para datos semi-estructurados, esquemas que cambian, y prototipado rápido

Herramientas de MongoDB

  • mongod — el servidor de base de datos
  • mongosh — shell interactiva para ejecutar comandos
  • MongoDB Compass — interfaz gráfica para explorar colecciones
  • MongoDB Atlas — servicio cloud gestionado

Formato de intercambio de datos ligero, legible por humanos y máquinas

BSON: Binary JSON

MongoDB almacena los documentos internamente en formato BSON

Representación binaria de JSON, más eficiente para almacenamiento y recorrido

BSON: Binary JSON

Soporta tipos adicionales que JSON no tiene:

  • ObjectId — identificador único de 12 bytes
  • Date — fechas con milisegundos
  • NumberInt, NumberLong, NumberDecimal
  • BinData — datos binarios
  • Regex — expresiones regulares

ObjectId

Cada documento tiene un campo _id que actúa como clave primaria

Si no se proporciona, MongoDB genera un ObjectId automáticamente

Es único globalmente, ordenable por tiempo de creación

Se genera de forma automática y funciona bien en sistemas distribuidos

Cada documento puede tener campos diferentes (schemaless)

Schemaless: flexibilidad

En MongoDB, la colección admite documentos con estructuras diferentes

En MongoDB, cada documento puede tener campos diferentes:

// Documento 1: tiene campo "notas"
{ nombre: "Jon", notas: [{asignatura: "ABD", nota: 7}] }

// Documento 2: no tiene "notas", pero tiene "becado"
{ nombre: "Miren", becado: true }

// Documento 3: tiene un campo "direccion" anidado
{ nombre: "Unai", direccion: { ciudad: "Bilbao" } }

Todos conviven en la misma colección sin problemas

find(): consultar documentos

db.alumnos.find()

db.alumnos.find({ edad: 21 })

db.alumnos.find({ nombre: "Ana García" })

// findOne() devuelve solo el primer resultado
db.alumnos.findOne({ nombre: "Ana García" })

Operadores de comparación

db.alumnos.find({ edad: { $gt: 21 } })

db.alumnos.find({ edad: { $gte: 20, $lte: 23 } })

db.alumnos.find({ edad: { $ne: 21 } })

db.alumnos.find({ edad: { $in: [20, 21, 22] } })

$gt, $gte, $lt, $lte, $eq, $ne, $in, $nin

Operadores lógicos

db.alumnos.find({
  $and: [
    { edad: { $gt: 20 } },
    { nombre: "Ana García" }
  ]
})

db.alumnos.find({
  $or: [
    { edad: { $lt: 20 } },
    { becado: true }
  ]
})

$and, $or, $not, $nor

AND implícito

// Cuando hay múltiples campos, MongoDB aplica AND:

// Esto:
db.alumnos.find({ edad: 21, nombre: "Ana García" })

// Es equivalente a esto:
db.alumnos.find({
  $and: [
    { edad: 21 },
    { nombre: "Ana García" }
  ]
})

Proyección (elegir campos)

db.alumnos.find(
  {},                          // filtro: todos
  { nombre: 1, email: 1 }     // proyección: solo estos campos
)

// Excluir campos (mostrar todos menos email):
db.alumnos.find(
  {},
  { email: 0 }
)

// El campo _id se incluye siempre por defecto
// Para excluirlo:
db.alumnos.find({}, { nombre: 1, _id: 0 })

1 = incluir, 0 = excluir. No se pueden mezclar (excepto _id)

Ordenar, limitar y saltar

db.alumnos.find().sort({ edad: 1 })

db.alumnos.find().sort({ edad: -1 })

db.alumnos.find().limit(5)

db.alumnos.find().skip(10).limit(5)

// Se pueden encadenar:
db.alumnos.find().sort({ edad: -1 }).limit(3)
// Los 3 alumnos más mayores

Contar documentos

db.alumnos.countDocuments()

db.alumnos.countDocuments({ edad: { $gt: 21 } })

// Contar con estimación rápida (para colecciones grandes):
db.alumnos.estimatedDocumentCount()

updateOne()

db.alumnos.updateOne(
  { nombre: "Ana García" },    // filtro
  { $set: { edad: 22 } }       // actualización
)

// $set modifica solo los campos indicados
// Si el campo no existe, lo crea:
db.alumnos.updateOne(
  { nombre: "Ana García" },
  { $set: { telefono: "666111222" } }
)

Operadores de actualización

// Incrementar un valor numérico
db.alumnos.updateOne(
  { nombre: "Jon Aguirre" },
  { $inc: { edad: 1 } }
)

// Eliminar un campo del documento
db.alumnos.updateOne(
  { nombre: "Unai López" },
  { $unset: { becado: "" } }
)

// Renombrar un campo
db.alumnos.updateOne(
  { nombre: "Ana García" },
  { $rename: { "telefono": "tlfno" } }
)

$set, $unset, $inc, $rename, $min, $max, $mul

updateMany()

db.alumnos.updateMany(
  { edad: { $gte: 20 } },
  { $set: { activo: true } }
)

// Resultado:
// { acknowledged: true,
//   matchedCount: 3,
//   modifiedCount: 3 }

updateOne modifica solo el primer documento que coincida

updateMany modifica todos los documentos que coincidan

Operaciones sobre arrays

// Añadir un elemento a un array
db.alumnos.updateOne(
  { nombre: "Jon Aguirre" },
  { $push: { notas: { asignatura: "SI", nota: 8.0 } } }
)

// Añadir varios elementos a un array
db.alumnos.updateOne(
  { nombre: "Jon Aguirre" },
  { $push: { notas: { $each: [
    { asignatura: "ED", nota: 6.5 },
    { asignatura: "SO", nota: 7.0 }
  ]}}}
)

// Eliminar un elemento de un array
db.alumnos.updateOne(
  { nombre: "Jon Aguirre" },
  { $pull: { notas: { asignatura: "ED" } } }
)

$push, $pull, $addToSet, $pop, $each

deleteOne() y deleteMany()

db.alumnos.deleteOne({ nombre: "Unai López" })

db.alumnos.deleteMany({ edad: { $lt: 20 } })

  // Borrar todos los documentos de la colección
db.alumnos.deleteMany({})

  // Eliminar la colección completa
db.alumnos.drop()

  // Eliminar la base de datos actual
db.dropDatabase()

replaceOne()

// Reemplaza un documento ENTERO (no campos individuales)
db.alumnos.replaceOne(
  { nombre: "Ana García" },
  {
    nombre: "Ana García Fernández",
    email: "ana.garcia@ehu.eus",
    edad: 22,
    grado: "Ingeniería Informática"
  }
)

Cuidado: replaceOne() sustituye todo el documento

A diferencia de updateOne() con $set, que modifica solo campos específicos

Consultas en documentos anidados

// Datos:
{ nombre: "Ana", direccion: { ciudad: "Bilbao", cp: "48001" } }
{ nombre: "Jon", direccion: { ciudad: "Donostia", cp: "20001" } }

// Buscar por campo anidado (dot notation):
db.alumnos.find({ "direccion.ciudad": "Bilbao" })

Dot notation: "campo.subcampo" para acceder a campos anidados

Consultas en arrays

// Datos:
{ nombre: "Jon", idiomas: ["euskera", "castellano", "inglés"] }
{ nombre: "Ana", idiomas: ["castellano", "francés"] }

// Buscar documentos que contengan "euskera" en el array:
db.alumnos.find({ idiomas: "euskera" })
// Resultado: Jon

// Buscar documentos con el array exacto:
db.alumnos.find({ idiomas: ["castellano", "francés"] })
// Resultado: Ana (orden importa)

// Buscar por tamaño del array:
db.alumnos.find({ idiomas: { $size: 3 } })
// Resultado: Jon (tiene 3 idiomas)

Consultas en arrays de objetos

// Datos:
{
  nombre: "Jon",
  notas: [
    { asignatura: "ABD", nota: 7.5 },
    { asignatura: "SI", nota: 8.0 }
  ]
}

// Buscar alumnos con nota > 7 en alguna asignatura:
db.alumnos.find({ "notas.nota": { $gt: 7 } })

// Buscar alumnos con nota > 7 en ABD específicamente:
db.alumnos.find({
  notas: {
    $elemMatch: {
      asignatura: "ABD",
      nota: { $gt: 7 }
    }
  }
})

$elemMatch: el mismo elemento del array debe cumplir todas las condiciones

$exists y $type

// Buscar documentos que TENGAN el campo "becado":
db.alumnos.find({ becado: { $exists: true } })

// Buscar documentos que NO tengan el campo "telefono":
db.alumnos.find({ telefono: { $exists: false } })

// Buscar documentos donde "edad" sea de tipo numérico:
db.alumnos.find({ edad: { $type: "number" } })

// Buscar documentos donde "nombre" sea un string:
db.alumnos.find({ nombre: { $type: "string" } })

Muy útiles en colecciones schemaless donde no todos los documentos tienen los mismos campos

Expresiones regulares

db.alumnos.find({ nombre: /^Ana/ })

db.alumnos.find({ email: /ehu\.eus$/ })

// Búsqueda case-insensitive:
db.alumnos.find({ nombre: { $regex: "garcia", $options: "i" } })

MongoDB soporta expresiones regulares PCRE (Perl Compatible Regular Expressions)

Aggregation Pipeline

Sistema de procesamiento de datos en etapas (stages)

Los documentos pasan por una cadena de transformaciones

Permite filtrar, transformar, agrupar y enriquecer datos paso a paso

db.coleccion.aggregate([
  { $etapa1: { ... } },
  { $etapa2: { ... } },
  { $etapa3: { ... } }
])

La salida de una etapa es la entrada de la siguiente

Etapas principales del Pipeline

Etapa Descripción
$match Filtra documentos
$group Agrupa y calcula
$project Selecciona/transforma campos
$sort Ordena resultados
$limit Limita resultados
$unwind Descompone arrays
$lookup Une colecciones

$match: filtrar

db.alumnos.aggregate([
  { $match: { edad: { $gt: 20 } } }
])

// La sintaxis del filtro es la misma que en find()
// $match debería ir lo antes posible para reducir
// el número de documentos que pasan a las siguientes etapas

$group: agrupar y calcular

// Datos en colección "pedidos":
{ cliente: "Ana", producto: "Monitor", precio: 250 }
{ cliente: "Jon", producto: "Teclado", precio: 45 }
{ cliente: "Ana", producto: "Ratón", precio: 25 }
{ cliente: "Jon", producto: "Monitor", precio: 250 }

db.pedidos.aggregate([
  {
    $group: {
      _id: "$cliente",
      total: { $sum: "$precio" },
      num_pedidos: { $sum: 1 }
    }
  }
])
// Resultado: { _id: "Ana", total: 275, num_pedidos: 2 }
//            { _id: "Jon", total: 295, num_pedidos: 2 }

Acumuladores de $group

MongoDB Descripción
$sum Suma valores o cuenta
$avg Media aritmética
$min Valor mínimo
$max Valor máximo
$first Primer valor del grupo
$push Array con todos los valores

Pipeline completo: ejemplo

db.pedidos.aggregate([
  { $match: { precio: { $gt: 30 } } },           // WHERE
  { $group: {                                      // GROUP BY
      _id: "$cliente",
      total: { $sum: "$precio" }
  }},
  { $match: { total: { $gt: 200 } } },            // HAVING
  { $sort: { total: -1 } }                         // ORDER BY
])

El pipeline permite construir consultas complejas paso a paso

$project: transformar campos

db.alumnos.aggregate([
  {
    $project: {
      nombre: 1,
      email_mayus: { $toUpper: "$email" },
      anio_nacimiento: {
        $subtract: [2026, "$edad"]
      },
      _id: 0
    }
  }
])
// Resultado:
// { nombre: "Ana García", email_mayus: "ANA@EHU.EUS",
//   anio_nacimiento: 2005 }

$unwind: descomponer arrays

// Dato: { nombre: "Jon", notas: [{asig: "ABD", nota: 7.5},
//                                 {asig: "SI", nota: 8.0}] }
// $unwind crea un documento por cada elemento del array:

db.alumnos.aggregate([
  { $unwind: "$notas" }
])
// Resultado:
// { nombre: "Jon", notas: { asig: "ABD", nota: 7.5 } }
// { nombre: "Jon", notas: { asig: "SI", nota: 8.0 } }

// Útil para después hacer $group por asignatura:
db.alumnos.aggregate([
  { $unwind: "$notas" },
  { $group: {
      _id: "$notas.asig",
      media: { $avg: "$notas.nota" }
  }}
])

$lookup: unión entre colecciones

// Si tuviéramos datos separados en dos colecciones:
// pedidos: { cliente_id: 1, producto: "Monitor", precio: 250 }
// clientes: { _id: 1, nombre: "Ana García" }

db.pedidos.aggregate([
  {
    $lookup: {
      from: "clientes",           // colección a unir
      localField: "cliente_id",    // campo en pedidos
      foreignField: "_id",         // campo en clientes
      as: "info_cliente"           // nombre del array resultado
    }
  }
])
// Resultado:
// { cliente_id: 1, producto: "Monitor", precio: 250,
//   info_cliente: [{ _id: 1, nombre: "Ana García" }] }

Diseño del esquema en MongoDB

Aunque MongoDB es schemaless, el diseño del esquema es fundamental

La pregunta clave: ¿embeber o referenciar?

  • Embeber: incluir los datos relacionados dentro del documento
  • Referenciar: guardar el _id del documento relacionado

La elección depende de patrones de acceso, tamaño y frecuencia de actualización

Embeber (desnormalizar)

// Toda la información en un solo documento
{
  nombre: "Ana García",
  direccion: {
    calle: "Gran Vía 1",
    ciudad: "Bilbao"
  },
  pedidos: [
    { producto: "Monitor", precio: 250, fecha: "2026-01-15" },
    { producto: "Ratón", precio: 25, fecha: "2026-02-20" }
  ]
}

Ventajas: lectura muy rápida (una sola consulta)

Inconvenientes: datos duplicados, documentos muy grandes (límite 16 MB)

Referenciar (normalizar)

// Colección "clientes"
{
  _id: ObjectId("aaa..."),
  nombre: "Ana García",
  direccion: { calle: "Gran Vía 1", ciudad: "Bilbao" }
}

// Colección "pedidos"
{
  cliente_id: ObjectId("aaa..."),
  producto: "Monitor",
  precio: 250,
  fecha: "2026-01-15"
}

Ventajas: sin duplicación, documentos más pequeños

Inconvenientes: requiere $lookup (más lento), múltiples consultas

¿Cuándo embeber? ¿Cuándo referenciar?

Criterio Embeber Referenciar
Relación 1:1, 1: pocos 1: muchos, N:M
Datos se leen juntos Sí No siempre
Datos cambian mucho No Sí
Tamaño subdatos Pequeño/limitado Grande/ilimitado
Prioridad Velocidad lectura Consistencia

Índices en MongoDB

Sin índices, MongoDB debe examinar todos los documentos (collection scan)

Un índice permite encontrar documentos sin escanear toda la colección

MongoDB crea automáticamente un índice sobre _id

Se pueden crear índices sobre cualquier campo o combinación de campos

Crear y gestionar índices

db.alumnos.createIndex({ email: 1 })   // 1 = ascendente

// Índice descendente:
db.alumnos.createIndex({ edad: -1 })

// Índice compuesto (sobre múltiples campos):
db.alumnos.createIndex({ nombre: 1, edad: -1 })

// Índice único (no permite duplicados):
db.alumnos.createIndex({ email: 1 }, { unique: true })

// Ver índices existentes:
db.alumnos.getIndexes()

// Eliminar un índice:
db.alumnos.dropIndex({ email: 1 })

explain(): analizar consultas

db.alumnos.find({ email: "ana@ehu.eus" }).explain("executionStats")

// Sin índice → COLLSCAN (escaneo completo):
// "winningPlan": { "stage": "COLLSCAN" }
// "totalDocsExamined": 10000

// Con índice → IXSCAN (escaneo de índice):
// "winningPlan": { "stage": "IXSCAN",
//                  "indexName": "email_1" }
// "totalDocsExamined": 1

Crear índices en campos que se usan frecuentemente en consultas

Los índices aceleran las lecturas pero ralentizan las escrituras

Índice de texto

// Crear un índice de texto para búsquedas full-text:
db.posts.createIndex({ contenido: "text", titulo: "text" })

db.posts.find({ $text: { $search: "MongoDB NoSQL" } })

// Buscar una frase exacta:
db.posts.find({ $text: { $search: "\"base de datos\"" } })

// Ordenar por relevancia:
db.posts.find(
  { $text: { $search: "MongoDB" } },
  { score: { $meta: "textScore" } }
).sort({ score: { $meta: "textScore" } })

Validación de esquema

Aunque MongoDB es schemaless, se puede definir validación opcional

Usar JSON Schema para definir reglas sobre los documentos

db.createCollection("alumnos", {
  validator: {
    $jsonSchema: {
      bsonType: "object",
      required: ["nombre", "email", "edad"],
      properties: {
        nombre: { bsonType: "string",
                   description: "debe ser un string" },
        email:  { bsonType: "string",
                   pattern: "^.+@.+$" },
        edad:   { bsonType: "int",
                   minimum: 16, maximum: 99 }
      }
    }
  }
})

Validación: niveles

// Nivel "strict" (por defecto): valida en insert y update
// Nivel "moderate": valida solo docs nuevos o válidos existentes

db.runCommand({
  collMod: "alumnos",
  validator: {
    $jsonSchema: {
      required: ["nombre", "email"],
      properties: {
        nombre: { bsonType: "string" },
        email: { bsonType: "string" }
      }
    }
  },
  validationLevel: "moderate",
  validationAction: "warn"    // "error" rechaza, "warn" avisa
})

Permite equilibrar flexibilidad de esquema e integridad de datos

Transacciones en MongoDB

Desde MongoDB 4.0, se soportan transacciones ACID multi-documento

const session = db.getMongo().startSession()
session.startTransaction()

try {
  const pedidosCol = session.getDatabase("tienda")
                            .getCollection("pedidos")
  const productosCol = session.getDatabase("tienda")
                              .getCollection("productos")

  pedidosCol.insertOne({ cliente: "Ana", total: 250 })
  productosCol.updateOne(
    { nombre: "Monitor" }, { $inc: { stock: -1 } }
  )

  session.commitTransaction()  // COMMIT
} catch (error) {
  session.abortTransaction()   // ROLLBACK
}

Replicación: Replica Set

MongoDB garantiza alta disponibilidad mediante Replica Sets

  • Primary: recibe todas las escrituras
  • Secondary(s): copias de los datos del Primary
  • Si el Primary cae, se elige automáticamente un nuevo Primary

En MongoDB viene integrado y es automático

Sharding: escalabilidad horizontal

Sharding = distribuir datos entre múltiples servidores

Cada servidor (shard) contiene un subconjunto de los datos

  • Permite manejar volúmenes de datos que no caben en un servidor
  • MongoDB distribuye automáticamente los datos según una shard key
  • Las consultas se distribuyen entre los shards relevantes

MongoDB: resumen

Característica Descripción
Modelo Documentos (JSON/BSON)
Esquema Flexible (schemaless con validación opcional)
Lenguaje API JavaScript (MQL)
Relaciones Embebido / referencias / $lookup
Escalabilidad Horizontal (sharding)
Transacciones ACID multi-documento (desde v4.0)
Fortalezas Flexibilidad, escalabilidad y buen encaje con datos semiestructurados

¿Cuándo usar MongoDB?

  • Datos semi-estructurados o con esquema variable
  • Catálogos de productos con atributos diferentes por categoría
  • Gestión de contenido (CMS, blogs)
  • Aplicaciones en tiempo real (chats, logs, IoT)
  • Prototipado rápido (no necesitas definir esquema)
  • Grandes volúmenes de datos que requieren escalabilidad horizontal

Buenas prácticas con MongoDB

  • Diseñar documentos según los patrones de consulta más frecuentes
  • Elegir entre embeber y referenciar con criterios de acceso y tamaño
  • Crear índices en campos usados en filtros y ordenaciones
  • Aplicar validación de esquema cuando la integridad sea crítica
  • Monitorizar consultas con explain() y métricas de rendimiento

from pymongo import MongoClient

client = MongoClient("mongodb://localhost:27017/")
db = client["tienda"]

# Pipeline de agregación en Python
pipeline = [
    {"$unwind": "$productos"},
    {"$group": {
        "_id": "$productos.nombre",
        "total": {"$sum": {
            "$multiply": ["$productos.precio",
                           "$productos.cantidad"]
        }}
    }},
    {"$sort": {"total": -1}},
    {"$limit": 5}
]

resultados = db.pedidos.aggregate(pipeline)
for r in resultados:
    print(f"{r['_id']}: {r['total']:.2f} €")
