IA simbólica vs IA estadística

IA simbólica: se aplica un modelo pre-existente construido por expertos que contiene conocimiento explícito codificado en símbolos (lógica)

IA estadística: el modelo aprende de datos de manera (semi)autónoma mediante procesos probabilísticos

Razonamiento

IA simbólica: lógico, determinista. Explicabilidad completa

IA estadística: aproximado, probabilístico. Baja explicabilidad

Datos

IA simbólica: necesita menos datos pero frágil ante el ruido

IA estadística: necesita muchos datos de alta calidad, más robusta ante el ruido

Requisitos

IA simbólica: expertos en el dominio

IA estadística: datos y alta capacidad de cómputo (GPUs)

RDF: modelo vs sintaxis

Ese modelo abstracto se puede representar con diferentes sintaxis: "Serializar" (escribir) en un archivo

No confundir el modelo con la sintaxis: RDF es más que un archivo XML (o JSON)

UPSERT (UPDATE INSERT)

Operación en SQL que actualiza si existe, inserta si no

INSERT INTO mascotas (id, nombre, color)

VALUES (2, "Fluffy", "negro")

ON DUPLICATE KEY UPDATE color = VALUES(color);

Objetivo: mantener un único valor actualizado para una fila

Basado en: Stratos Kontopoulos (Medium)

En RDF cargar datos es aditivo

Al cargar datos las nuevas tripletas se añaden

Ejemplo: añadir ex:color "negro" no elimina ex:color "marrón"

Resultado: el recurso puede tener múltiples valores para la misma propiedad

Sin semántica de "valor único"

El modelo RDF permite varios valores por propiedad en un recurso

Incluso con owl:FunctionalProperty (OWL), la restricción es lógica: la Triple Store no borra automáticamente

Los Triple Stores pueden permitir, avisar (SHACL) o inferir (OWL); no borran automáticamente

Open World Assumption (OWA)

En RDF (y OWL), que falte un dato no implica que sea falso u obsoleto (No tener información no implica negación, ya que puede haber información en otro lugar o en el futuro en internet)

Para sustituir un valor, hay que indicar el borrado explícito

Opciones: SPARQL Update (DELETE/INSERT) o reemplazar el grafo completo

Linked Data

Publicación de datos estructurados en la web de forma que puedan ser enlazados e interpretados por máquinas

Basado en estándares web: URI, HTTP(S), RDF, SPARQL

Propuesto por Tim Berners-Lee (Inventor de la Web) en 2006

Principios de Linked Data

  1. Usar URIs para identificar recursos
  2. Usar URIs HTTP(S) para acceder a esos recursos
  3. Cuando se accede a un recurso, proporcionar sus datos usando estándares (RDF, SPARQL)
  4. Incluir enlaces a otras URIs para descubrir más recursos

https://www.w3.org/DesignIssues/LinkedData.html

Linked Data: Negociación de contenido

Una URI puede devolver diferentes representaciones según el cliente

Navegador web → HTML (para humanos)

Cliente RDF → RDF/XML, Turtle, JSON-LD (para máquinas)

Mediante HTTP Content Negotiation (cabecera Accept)
