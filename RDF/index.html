Linked Open Vocabularies

RDF: Blank Nodes

Nodos anónimos sin URI: recursos sin identificador global

Se usan cuando no necesitamos referenciar el recurso desde fuera

Sintaxis Turtle: _:b1, _:b2, ... o []

Problema: no se pueden enlazar desde otros grafos (≠ Linked Data)

RDF: Blank Nodes - Ejemplo

@prefix ex: <http://example.org/> .
@prefix foaf: <http://xmlns.com/foaf/0.1/> .

ex:Mikel foaf:knows [
    foaf:name "Jon" ;
    foaf:mbox <mailto:jon@example.org>
] .

El recurso "Jon" no tiene URI propia

RDF: Reificación

Hacer afirmaciones sobre afirmaciones (meta-triples)

Necesario para expresar: confianza, procedencia, temporalidad

Tradicionalmente: rdf:Statement, rdf:subject, rdf:predicate, rdf:object

RDF 1.1: Named Graphs (más eficiente)

RDF: Reificación - Ejemplo clásico

@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix ex: <http://example.org/> .

ex:statement1 rdf:type rdf:Statement ;
    rdf:subject ex:Mikel ;
    rdf:predicate ex:livesIn ;
    rdf:object ex:Bilbao ;
    ex:confidence "0.9"^^xsd:float .

RDF: Colecciones

Listas ordenadas: rdf:List, rdf:first, rdf:rest, rdf:nil

Sintaxis Turtle abreviada: ( elemento1 elemento2 elemento3 )

Contenedores desordenados: rdf:Bag, rdf:Seq, rdf:Alt

Uso moderno: usar predicados específicos del dominio

RDF: Colecciones - Ejemplo

@prefix ex: <http://example.org/> .

ex:Mikel ex:favoriteColors (
    ex:Blue
    ex:Green
    ex:Red
) .

Equivale a una lista enlazada en RDF

RDF: Internacionalización

Literales con etiquetas de idioma (BCP 47)

"Hello"@en, "Hola"@es, "Kaixo"@eu

Permite datos multilingües en el mismo grafo

Fundamental para datos globales y open data

RDF: Internacionalización - Ejemplo

@prefix ex: <http://example.org/> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .

ex:Bilbao rdfs:label "Bilbao"@es ;
    rdfs:label "Bilbo"@eu ;
    rdfs:label "Bilbao"@en .

RDF: IRIs vs URIs

IRI: Internationalized Resource Identifier

Extensión de URI que permite caracteres Unicode

http://example.org/东京 (Tokio en chino)

RDF 1.1 usa IRIs, no solo URIs ASCII

RDF: Vocabulario RDFS

rdfs:Class - definir clases

rdfs:subClassOf - jerarquías de clases

rdfs:Property - definir propiedades

rdfs:subPropertyOf - jerarquías de propiedades

rdfs:domain, rdfs:range - restricciones de propiedades

rdfs:label, rdfs:comment - anotaciones

RDF: RDFS - Ejemplo

@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix ex: <http://example.org/> .

ex:Person rdfs:subClassOf ex:Agent .
ex:Student rdfs:subClassOf ex:Person .

ex:estudiaEn rdfs:domain ex:Student ;
    rdfs:range ex:Universidad .

RDF Star (RDF-star)

Extensión reciente de RDF: permitir triples como sujetos u objetos

Sintaxis: << sujeto predicado objeto >>

Simplifica la reificación (más intuitivo que Named Graphs)

Estándar W3C en desarrollo (2024)

RDF Star - Ejemplo

@prefix ex: <http://example.org/> .

<< ex:Mikel ex:livesIn ex:Bilbao >> 
    ex:since "2020"^^xsd:gYear ;
    ex:confidence "high" .

RDF: Buenas prácticas

Usar URIs HTTP dereferenciales (303 redirect o Hash URIs)

Reutilizar vocabularios existentes antes de crear nuevos

Proporcionar labels en múltiples idiomas (rdfs:label)

Documentar vocabularios con rdfs:comment

Evitar blank nodes en datos Linked Open Data

RDF: Patrones de diseño

N-ary relations: usar recursos intermedios para relaciones complejas

Qualified relations: añadir contexto a relaciones

Versioning: usar Named Graphs para versiones temporales

W3C: Defining N-ary Relations

RDF: Validación de datos

SHACL (Shapes Constraint Language): validar estructura de grafos RDF

ShEx (Shape Expressions): alternativa a SHACL, más concisa

Necesario para garantizar calidad de datos en producción

Diferente de OWL: validación vs inferencia

RDF: Serialización N-Quads

Extensión de N-Triples para Named Graphs

Cada línea: <sujeto> <predicado> <objeto> <grafo> .

Formato simple para exportar/importar múltiples grafos

Usado en dumps de grandes bases de datos (DBpedia, Wikidata)

RDF: JSON-LD

Serialización RDF en JSON: fácil para desarrolladores web

Usado por Schema.org para metadatos en páginas web

Google, Bing, Yahoo lo procesan para mejorar búsquedas

@context define el mapeo de claves JSON a URIs RDF

RDF: JSON-LD - Ejemplo

{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "Mikel Egaña",
  "jobTitle": "Profesor",
  "worksFor": {
    "@type": "Organization",
    "name": "UPV/EHU"
  }
}

Linked Data

Publicación de datos estructurados en la web de forma que puedan ser enlazados e interpretados por máquinas

Basado en estándares web: URIs, HTTP, RDF

Propuesto por Tim Berners-Lee en 2006

Principios de Linked Data

  1. Usar URIs para identificar recursos
  2. Usar URIs HTTP para que se puedan buscar
  3. Proporcionar información útil usando estándares (RDF, SPARQL)
  4. Incluir enlaces a otras URIs para descubrir más recursos

https://www.w3.org/DesignIssues/LinkedData.html

Linked Data: Negociación de contenido

Una URI puede devolver diferentes representaciones según el cliente

Navegador web → HTML (para humanos)

Cliente RDF → RDF/XML, Turtle, JSON-LD (para máquinas)

Mediante HTTP Content Negotiation (cabecera Accept)

Linked Data: Ejemplo Wikidata

URI del recurso: http://www.wikidata.org/entity/Q8692

Página web (HTML): https://www.wikidata.org/wiki/Q8692

Datos RDF: Agregar .ttl, .rdf, .json al final de la URI

Linked Data: Ejemplo DBpedia

URI del recurso: http://dbpedia.org/resource/Bilbao

Página web: https://dbpedia.org/page/Bilbao

Datos RDF: https://dbpedia.org/data/Bilbao

Endpoints SPARQL públicos permiten consultas federadas

Linked Data: 5 estrellas

★ Datos disponibles en la web (cualquier formato) con licencia abierta

★★ Datos estructurados y legibles por máquina (ej. Excel en vez de imagen)

★★★ Formato no propietario (ej. CSV en vez de Excel)

★★★★ Usar estándares W3C (RDF, SPARQL) para identificar recursos

★★★★★ Enlazar tus datos con otros datos (contexto)
